Enforcement

Consistency enforced — not requested

Controls that stop overrides and prevent regressions.

1. Strict naming

Token paths are fixed. No ad-hoc names or synonyms.

Prevents token sprawl and lookup errors.

Allowed: color.text.primary
Blocked: darkText, textMain
Impact: fewer orphan tokens → faster fixes

2. Token access rules

Components may only consume semantic tokens — primitives are locked.

Removes inline overrides and code-level drift.

Primitive: #111111
Semantic: color.action.primary.surface
Impact: fewer runtime mismatches → lower QA load

3. Component assembly

Components are assembled from component tokens — no custom values inside components.

Finite variants only — no one-offs.

Button.size → size.control
Button.padding → space.control
Impact: fewer hotfixes after releases

4. Modes & Policy

Modes (policy) rebind semantic tokens — density, color scheme and platform mappings live here.

Policy changes map semantics to primitives (auditable and reversible).

Token: text.primary
Light → #111111 · Dark → #FFFFFF
Impact: consistent behavior across modes and platforms

5. Change control

Semantic changes require review, versioning, and a visible changelog; breaking changes are gated.

Safe evolution with traceable history.

Allowed: reviewed semantic change
Blocked: silent primitive swaps / component overrides
Impact: auditable changes → predictable releases
